# Keyword Extraction Quality Design

**Date:** 2026-04-07

## Goal

keyword page 생성 품질을 높이기 위해 본문 전체 빈도 중심 추출을 `제목 시드 + 구조 신호 + 본문 2차 승격` 방식으로 바꾸고, 삼성 DS 관련 도메인 오인식을 코드와 프롬프트 양쪽에서 막는다.

## Problems To Fix

- `개요`, `체크리스트`, `문서` 같은 구조어가 keyword로 올라온다.
- 제목이 `회의록`, `공지`, `정리`처럼 약한 경우 핵심 주제가 잘 뽑히지 않는다.
- 제목에 없는 중요 키워드가 본문에 있어도 충분히 승격되지 않는다.
- `삼성 DS`가 `디스플레이`로 오인식될 위험이 있다.
- 문서 길이에 따라 너무 적은 keyword만 생성된다.

## Approach

### 1. Domain normalization

- 코드 레벨 정규화 사전을 둔다.
- `삼성 DS`, `삼성DS`, `DS부문`, `Device Solutions`, `device solutions`는 모두 `DS부문`으로 통합한다.
- 문서에 `디스플레이`, `삼성디스플레이`, `display`가 직접 명시되지 않은 한 `DS부문`을 디스플레이 의미로 바꾸지 않는다.
- LLM 프롬프트에도 같은 금지 규칙을 명시한다.

### 2. Weighted keyword extraction

- 제목에서 1차 keyword seed를 뽑는다.
- 제목이 약하면 `H1/H2/H3`, 표 헤더, 리스트 제목, 링크 텍스트를 먼저 본다.
- 일반 본문은 2차 확장용으로만 사용한다.
- 구조별 가중치:
  - 제목 `5`
  - 소제목 `4`
  - 표 헤더 `3`
  - 링크 텍스트 `2`
  - 일반 본문 `1`

### 3. Title blacklist and structural stopwords

- 제목이나 구조 신호에서 추출되더라도 버릴 단어를 늘린다.
- 예: `회의록`, `공지`, `자료`, `정리`, `개요`, `체크리스트`, `위키`, `문서`, `링크`, `화면`
- 구조용 HTML 태그와 markdown artefact는 계속 제거한다.

### 4. Secondary promotion

- 제목에 없는 토큰이라도 아래 조건이면 keyword 후보로 승격한다.
- 일반 본문에서 3회 이상 등장
- 그리고 소제목/표 헤더/링크 텍스트 중 하나에서 1회 이상 등장
- 또는 기존 keyword page와 같은 섹션에서 반복적으로 같이 등장

### 5. Minimum keyword counts

- 문서 길이에 따라 최소 keyword 수를 강제한다.
- 사용자가 요청한 대로 이전 설계의 3배 기준을 쓴다.
- 기준:
  - 150자 이하: 최소 `3`
  - 151~600자: 최소 `6`
  - 601~2000자: 최소 `9`
  - 2001~5000자: 최소 `12`
  - 5001자 이상: 최소 `15`
- 점수 컷을 단계적으로 낮춰 최소 개수까지 채우되, blacklist/stopword는 절대 넣지 않는다.

### 6. Output behavior

- 문서당 대표 keyword는 최소 개수 이상 연결한다.
- 최대 개수는 별도 제한하지 않는다.
- 다만 같은 keyword page에 지나치게 일반적인 단어가 쌓이지 않도록 점수 순서를 유지한다.

### 7. Testing

- `삼성 DS` 제목/본문이 `DS부문`으로 정규화되고 `디스플레이` keyword가 생기지 않는지 테스트한다.
- `회의록` 같은 약한 제목에서도 구조 신호 기반 keyword가 추출되는지 테스트한다.
- 긴 문서에서 최소 keyword 수가 3배 기준으로 보장되는지 테스트한다.
- 기존 demo seed에서 구조어 keyword가 다시 생기지 않는지 테스트한다.
